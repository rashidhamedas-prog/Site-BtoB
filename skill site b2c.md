```markdown\# Project Blueprint: Premium Single-Brand Fashion E-Commerce (Iran Market)\## 1. Project Overview\*\*Goal:\*\* Build the most complete, flawless, high-performance, single-brand clothing e-commerce platform in Iran. The UX/UI standard should match Digikala, but heavily optimized for a fashion brand.\*\*Target Audience:\*\* Iranian consumers (Persian language, RTL layout, local payment/shipping).\*\*Architecture:\*\* Custom coding (No WordPress). Headless architecture (Next.js + Node.js).\## 2. Tech Stack\*   \*\*Frontend:\*\* Next.js (App Router) + Tailwind CSS (RTL) + Zustand (State).\*   \*\*Backend:\*\* Node.js (NestJS) + TypeScript.\*   \*\*Database:\*\* PostgreSQL (Prisma ORM) + Redis (Caching/Cart).\*   \*\*Queue:\*\* BullMQ (For async tasks like feed generation \& postbacks).\## 3. Database Schema (Core Entities)\*   \*\*Product:\*\* `id`, `name`, `description`, `category\_id`, `collection\_id`, `is\_pre\_order`, `pre\_order\_date`, `tags` (New, BestSeller, Sale).\*   \*\*ProductVariant:\*\* `id`, `product\_id`, `color`, `size`, `sku`, `price`, `discount\_price`, `stock\_quantity`, `images\_array`. (Stock managed strictly at this level).\*   \*\*Category:\*\* `id`, `parent\_id`, `name`, `slug`.\*   \*\*Collection:\*\* `id`, `name`, `season` (e.g., Spring 2024).\*   \*\*User:\*\* `id`, `phone\_number`, `first\_name`, `last\_name`, `national\_code`, `wallet\_balance`.\*   \*\*Address:\*\* `id`, `user\_id`, `province`, `city`, `postal\_code`, `street\_details`.\*   \*\*Order:\*\* `id`, `user\_id`, `address\_id`, `total\_amount`, `shipping\_cost`, `status`, `payment\_method`, `tracking\_code`, `affiliate\_id`.\*   \*\*OrderItem:\*\* `id`, `order\_id`, `variant\_id`, `quantity`, `price\_at\_purchase`.\*   \*\*ReturnRequest (RMA):\*\* `id`, `order\_item\_id`, `reason`, `requested\_size` (for exchange), `status`, `refund\_type`.\## 4. Frontend Features (Flawless Clothing UX/UI)\*   \*\*Layout:\*\* Fully RTL, Vazirmatn font, Mobile-first.\*   \*\*Navigation:\*\* Mega Menu with category images (Men, Women, Kids -> Shirts, Pants -> Collections).\*   \*\*Product Listing Page (PLP):\*\*&#x20;   \*   Ajax infinite scroll.&#x20;   \*   Advanced Clothing Filters: Size, Color, Price, Fabric Type, Collar Type, Collection.\*   \*\*Product Detail Page (PDP) - \*Critical for Clothing\*:\*\*&#x20;   \*   Image gallery with hover-zoom and short video support of the model wearing it.&#x20;   \*   Variant selection: Color swatches and Size buttons. Automatically grey out unavailable sizes.&#x20;   \*   \*\*Size Chart Modal:\*\* Dynamic table per product showing exact measurements (S, M, L chest/width/length).&#x20;   \*   \*\*Model Info Section:\*\* Display model's height and worn size (e.g., "Model is 180cm wearing size M").&#x20;   \*   \*\*Pre-order Badge:\*\* Show countdown or arrival date if `is\_pre\_order` is true.&#x20;   \*   Ajax "Add to Cart" (no reload).\*   \*\*Cart \& Checkout (Single Page):\*\*&#x20;   \*   Ajax Cart drawer.&#x20;   \*   \*\*Cross-sell in Cart:\*\* "Complete your look" (suggest matching pants for a shirt).&#x20;   \*   Auto-complete address (Integration with Iran postcode/standard API).&#x20;   \*   Real-time shipping cost calculation (Weight/Province based: Pishtaz, Tipax, Chapar, Tehran Bike Courier).&#x20;   \*   Payment options: Zarinpal, IDPay, COD (Cash on Delivery with validation).\*   \*\*User Dashboard:\*\*&#x20;   \*   OTP Login (SMS).&#x20;   \*   Order Tracking (Visual timeline: Processing -> Shipped -> Delivered + API tracking).&#x20;   \*   \*\*Advanced RMA System:\*\* User clicks "Return/Exchange", selects item, chooses reason ("Size too small/big"), and requests a new size. Admin processes this from the panel.&#x20;   \*   \*\*Digital Wallet:\*\* Refunds for returns can be credited to the wallet (with an optional bonus percentage to encourage wallet credit over bank refunds).&#x20;   \*   Wishlist.\## 5. Deep Integration with Sales-Boost Platforms (Crucial)Build a centralized `Integrations Module`. The system must automatically format site data precisely according to each platform's rules.\### 5.1. Price Comparison Engines (Torob \& Bam)\*   \*\*Torob Integration:\*\*&#x20;   \*   Endpoint: `/feeds/torob.xml`&#x20;   \*   System must map DB fields exactly to Torob's schema: `<product\_id>`, `<title>`, `<price>`, `<old\_price>`, `<availability>`, `<image\_link>`, `<link>`, `<category>`, `<brand>`.&#x20;   \*   Must include Torob's clothing attributes: `<sizes>`, `<colors>`.\*   \*\*Bam Integration:\*\*&#x20;   \*   Endpoint: `/feeds/bam.csv`&#x20;   \*   Same mapping logic, formatted as CSV per Bam's guidelines.\### 5.2. Affiliate \& CPA Networks (Yektanet, Affer, Afsona)\*   \*\*Admin Panel UI:\*\* Settings page to input `Advertiser ID` and `Pixel Codes`.\*   \*\*Frontend Tracking:\*\* Inject scripts into `<head>`. Fire `PageView` and `AddToCart` events.\*   \*\*Backend Postback (S2S):\*\* When an order status changes to "Paid", the backend must send an HTTP POST/GET request to the respective network's Postback URL with `order\_id`, `amount`, and `affiliate\_id`.\## 6. Backend \& Admin Panel Features\*   \*\*Dashboard:\*\* Sales analytics today, low stock alerts (at variant level), best-selling sizes.\*   \*\*Inventory:\*\* Strict variant-level stock management.\*   \*\*Order Management:\*\* Update status, trigger SMS notifications (Kavenegar/Farapayamak), issue electronic invoices.\*   \*\*RMA Management:\*\* Process returns, approve size exchanges, update wallet balances.\*   \*\*Roles (RBAC):\*\* Super Admin, Warehouse (stock only), Support (orders/users), Content (blog/products).\*   \*\*Fashion Blog (CMS):\*\* Full editor for fashion articles to boost SEO (internal linking to products).\## 7. SEO \& Performance Requirements\*   \*\*Technical SEO:\*\* Next.js SSR/SSG for all product, category, and blog pages.\*   \*\*Schema Markup:\*\* `Product`, `Offer`, `BreadcrumbList`, `AggregateRating` JSON-LD.\*   \*\*Performance:\*\* WebP image optimization, Redis caching for DB queries, CDN for static assets.\*   \*\*URL Structure:\*\* Clean URLs (e.g., `/men-shirts/blue-cotton-shirt`).\## 8. Implementation Instructions for AI (Cursor)\*   \*\*Phase 1:\*\* Initialize Next.js frontend and NestJS backend. Setup Prisma schema with exact relations.\*   \*\*Phase 2:\*\* Build OTP Auth and User Dashboard (including Wallet \& RMA history).\*   \*\*Phase 3:\*\* Build Product, Category, Variant, and Collection logic (CRUD). Ensure stock logic is perfectly mapped to variants.\*   \*\*Phase 4:\*\* Develop the Storefront (PLP with filters, PDP with Size Chart \& Model Info, Ajax Cart, Cross-sell).\*   \*\*Phase 5:\*\* Implement Checkout, Zarinpal Gateway, and Shipping APIs.\*   \*\*Phase 6:\*\* Implement the advanced RMA (Return/Exchange size) workflow.\*   \*\*Phase 7:\*\* Build the Integrations Module. Create dynamic XML/CSV feed generators for Torob/Bam. Implement S2S Postback logic for Yektanet/Affer.\*\*Rule:\*\* Write clean, modular, well-commented code. Use TypeScript strictly. All UI text must be in Persian.```
